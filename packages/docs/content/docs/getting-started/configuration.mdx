The Citric collector uses a standard OpenTelemetry Collector configuration file. This page covers the key sections you need to configure.

## Minimal configuration

A minimal setup with the GitHub Actions receiver and ClickHouse exporter:

```yaml
receivers:
  githubactions:
    endpoint: 0.0.0.0:3333
    path: /webhook/github
    secret: your-webhook-secret

exporters:
  clickhouse:
    endpoint: 'http://localhost:8123'
    username: 'default'
    database: 'default'

service:
  pipelines:
    traces:
      receivers: [githubactions]
      exporters: [clickhouse]
    logs:
      receivers: [githubactions]
      exporters: [clickhouse]
```

## Full configuration reference

Below is an annotated version of the full configuration with all available options.

### Receiver

The `githubactions` receiver accepts webhook events from GitHub Actions:

```yaml
receivers:
  githubactions:
    # Address and port to listen on (required)
    endpoint: 0.0.0.0:3333
    # URL path for the webhook endpoint
    path: /webhook/github
    # Webhook secret for payload validation (recommended)
    secret: your-webhook-secret
    # GitHub API config — required for fetching workflow logs
    gh_api:
      auth:
        app_id: 1234
        private_key_path: /path/to/key.pem
```

See [GitHub Actions integration](/docs/ci-integration/github-actions) for details on setting up authentication.

### Exporters

#### ClickHouse

```yaml
exporters:
  clickhouse:
    endpoint: 'http://localhost:8123'
    username: 'default'
    database: 'default'
```

#### Debug

Useful during initial setup to verify data is flowing:

```yaml
exporters:
  debug:
    verbosity: detailed
```

### Connectors

Connectors transform data between pipelines. The `spanmetrics` connector generates metrics from trace spans, and `testlogstotraces` parses test output from logs into trace spans.

#### spanmetrics

Generates duration metrics from trace spans:

```yaml
connectors:
  spanmetrics:
    histogram:
      explicit:
        buckets: [5s, 15s, 30s, 1m, 5m, 10m, 30m]
      unit: s
    aggregation_temporality: 'AGGREGATION_TEMPORALITY_CUMULATIVE'
    exemplars:
      enabled: true
```

#### testlogstotraces

Parses verbose test output from CI logs and generates test trace spans. Required for [test analytics](/docs/test-analytics).

```yaml
connectors:
  testlogstotraces:
```

No additional configuration is needed — the connector auto-detects Go test and Vitest output formats.

### Service pipelines

Pipelines define how data flows through the collector:

```yaml
service:
  pipelines:
    traces:
      receivers: [githubactions]
      exporters: [spanmetrics, clickhouse]
    logs:
      receivers: [githubactions]
      exporters: [testlogstotraces, clickhouse]
    traces/tests:
      receivers: [testlogstotraces]
      exporters: [clickhouse]
    metrics:
      receivers: [spanmetrics]
      exporters: [clickhouse]
```

Key points:
- The `traces` pipeline receives webhook events and exports trace spans
- The `logs` pipeline fetches workflow logs and can feed them to `testlogstotraces` for test parsing
- The `metrics` pipeline receives computed span metrics from the `spanmetrics` connector

## GitHub API authentication

The receiver needs GitHub App API access to fetch workflow logs.

### GitHub App (recommended)

GitHub Apps have higher rate limits and finer-grained permissions. You need:

- `app_id` — from the GitHub App settings page
- `private_key_path` — path to the `.pem` file generated when creating the app

```yaml
gh_api:
  auth:
    app_id: 1234
    private_key_path: /path/to/key.pem
```

See [GitHub Actions — GitHub App setup](/docs/ci-integration/github-actions#github-app-recommended) for step-by-step instructions.

## GitHub Enterprise

For GitHub Enterprise, configure the base URL:

```yaml
gh_api:
  base_url: https://github.example.com/api/v3
  upload_url: https://github.example.com/api/uploads
  auth:
    # ...
```
