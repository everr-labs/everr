Citric is a CI/CD observability platform built on OpenTelemetry. It receives webhook events from your CI provider, processes them into OpenTelemetry traces and logs, stores them in ClickHouse, and provides a web dashboard for visualization and analysis.

## How it works

```
GitHub Actions ──webhook──▶ Collector ──traces/logs──▶ ClickHouse ──queries──▶ Dashboard
```

1. Your CI provider sends webhook events (workflow runs, job completions) to the Citric collector
2. The collector transforms these events into OpenTelemetry traces and logs
3. Telemetry data is exported to ClickHouse for storage
4. The web dashboard queries ClickHouse to visualize pipeline performance, test results, and more

## Prerequisites

- **ClickHouse** — for storing telemetry data
- **PostgreSQL** — for dashboard application data
- **Docker** (recommended) or **Go 1.25+** — for running the collector
