The Citric dashboard is a web application that queries ClickHouse and displays your CI/CD telemetry data — pipeline runs, job durations, test results, and more.

## Infrastructure

Citric requires ClickHouse and PostgreSQL. The easiest way to run them is with Docker Compose. From the project root:

```bash
docker compose up -d
```

This starts:

- **ClickHouse** on port `8123` — stores OpenTelemetry traces, logs, and metrics
- **PostgreSQL** on port `5432` — stores dashboard application data (users, settings)

## Environment variables

Create a `.env` file in `packages/app/` based on `.env.example`:

```bash
cp packages/app/.env.example packages/app/.env
```

| Variable | Description | Default |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://postgres:postgres@localhost:5432/citric` |
| `CLICKHOUSE_HOST` | ClickHouse server hostname | `localhost` |
| `CLICKHOUSE_PORT` | ClickHouse HTTP port | `8123` |
| `CLICKHOUSE_USERNAME` | ClickHouse username | `citric` |
| `CLICKHOUSE_PASSWORD` | ClickHouse password | `citric` |
| `CLICKHOUSE_DATABASE` | ClickHouse database name | `default` |

## Running the dashboard

Install dependencies and start the development server:

```bash
pnpm install
pnpm dev
```

The dashboard will be available at [http://localhost:5173](http://localhost:5173).
