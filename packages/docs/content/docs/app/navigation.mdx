Everr organizes CI/CD analysis into focused dashboard areas.

## Main sections

- **Overview** (`/`): high-level health, success rate, recent runs, top failing jobs/workflows.
- **Runs** (`/runs`): search and filter individual workflow runs.
- **Analytics** (`/analytics`): performance trends across workflows.
- **Cost Analysis** (`/cost-analysis`): estimated CI cost by OS, runner, repo, and workflow.
- **Failures** (`/failures`): recurring failure patterns and failure trends.
- **Flaky Tests** (`/flaky-tests`): unstable tests with failure rates and history.
- **Tests Overview** (`/tests-overview`): aggregate test health, duration, and failures with drill-down.
- **Repositories** (`/repos`): repository-level CI insights.
- **Workflows** (`/workflows`): workflow-specific metrics and recent runs.

## Recommended usage pattern

1. Start at **Overview** for current health.
2. Jump to **Runs** or **Failures** when regressions appear.
3. Use **Tests Overview** for test reliability and performance analysis.
4. Use **Cost Analysis** to spot expensive pipelines.

For hands-on triage flows, see [App Workflows](/docs/app/workflows).
