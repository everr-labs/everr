Citric can parse verbose test output from your CI workflow logs and generate OpenTelemetry trace spans for each test. This enables test result tracking, flaky test detection, failure analysis, and duration trends â€” all from data you're already producing.

## How it works

1. Your CI workflows run tests with **verbose output** enabled
2. The Citric collector fetches workflow logs via the GitHub API
3. The `testlogstotraces` connector parses the log output and identifies test results
4. Each test becomes a span with attributes like name, result, duration, and framework
5. The dashboard surfaces this data through the [Test Results](/docs/features/test-results), [Flaky Tests](/docs/features/flaky-tests), and [Failure Analysis](/docs/features/failure-analysis) features

## Supported frameworks

| Framework | Flag | Status |
|-----------|------|--------|
| [Go tests](/docs/test-analytics/go-tests) | `go test -v` | Supported |
| [Vitest](/docs/test-analytics/vitest) | `--reporter=verbose` | Supported |

More test frameworks will be supported in the future.

## Requirements

The key requirement is that **tests must produce verbose output** so the collector can parse individual test results. Standard (non-verbose) test output only reports summary pass/fail and cannot be parsed into per-test spans.

## Collector configuration

Add the `testlogstotraces` connector to your collector config:

```yaml
connectors:
  testlogstotraces:

service:
  pipelines:
    logs:
      receivers: [githubactions]
      exporters: [testlogstotraces, clickhouse]
    traces/tests:
      receivers: [testlogstotraces]
      exporters: [clickhouse]
```

The connector auto-detects Go test and Vitest output formats â€” no additional configuration is needed.

## Span attributes

Each test span includes the following attributes:

| Attribute | Description |
|-----------|-------------|
| `citric.test.name` | Test name (e.g. `TestParseConfig` or `renders login form`) |
| `citric.test.result` | Result: `pass`, `fail`, or `skip` |
| `citric.test.duration_seconds` | Test execution duration in seconds |
| `citric.test.framework` | Test framework (`go` or `vitest`) |
| `citric.test.package` | Go package path or Vitest file path |
| `citric.test.is_subtest` | Whether this is a subtest/nested describe |
| `citric.test.parent_test` | Parent test name for subtests |
