---
title: Set Up Local Telemetry
description: Point a local app or command at the Everr CLI collector.
---

Use local telemetry when you want traces, logs, or errors from a service running on your machine.

The Everr CLI starts a local OpenTelemetry collector. Your app sends OTLP/HTTP telemetry to that collector, and you query the data with `everr local query`.

See [Local Telemetry](/docs/cli/telemetry) for collector startup and endpoint commands.

## 1. Capture runtime errors

Runtime errors should be sent as structured OpenTelemetry logs, not only printed to the terminal.

For Node.js apps:

- record `uncaughtException` and `unhandledRejection`
- export error logs to `/v1/logs`
- flush the SDK before the process exits
- preserve the original crash behavior

For browser apps:

- record `window.error` and `window.unhandledrejection`
- export error logs to `/v1/logs`
- include useful attributes like route, URL, exception type, message, and stacktrace
- redact tokens, passwords, emails, and request bodies before export

## 2. Query fresh telemetry

After triggering the local code path, check recent traces:

```bash
everr local query "SELECT Timestamp, ServiceName, SpanName, Duration, StatusCode FROM otel_traces WHERE Timestamp > now() - INTERVAL 10 MINUTE ORDER BY Timestamp DESC LIMIT 20"
```

Check recent logs and errors:

```bash
everr local query "SELECT Timestamp, ServiceName, SeverityText, Body FROM otel_logs WHERE Timestamp > now() - INTERVAL 10 MINUTE ORDER BY Timestamp DESC LIMIT 20"
```

If you expect a specific service, filter by `ServiceName`:

```bash
everr local query "SELECT Timestamp, ServiceName, Body FROM otel_logs WHERE ServiceName = 'my-local-app' AND Timestamp > now() - INTERVAL 10 MINUTE ORDER BY Timestamp DESC LIMIT 20"
```

## Use `everr wrap` for plain commands

Use `everr wrap` when a command is not OpenTelemetry-instrumented, but you still want its stdout and stderr in local logs:

```bash
everr wrap -- npm run lint
```

This is useful for build, lint, and test commands. For a runtime service or web app, prefer real OpenTelemetry instrumentation instead.

`everr wrap` requires the local collector to be running. If the collector is unavailable, the wrapped command is not run.

## Troubleshooting

If no rows appear:

- run `everr local status`
- confirm your app uses the URL from `everr local endpoint`
- use OTLP over HTTP, not browser gRPC
- set `OTEL_SERVICE_NAME` so rows are easy to find
- trigger the instrumented code path again
- query both `otel_traces` and `otel_logs`
