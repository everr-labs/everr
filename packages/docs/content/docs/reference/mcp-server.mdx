Citric includes a built-in [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that exposes CI/CD observability data as tools for AI coding assistants. This enables agents in Claude Code, Cursor, and other MCP-compatible clients to query pipeline health, diagnose failures, analyze test flakiness, and more — directly from your IDE.

## Endpoint

The MCP server is available at:

```
POST /api/mcp
```

It implements the [Streamable HTTP transport](https://modelcontextprotocol.io/specification/2025-11-25/basic/transports#streamable-http) in stateless mode — each request is independent with no session tracking.

## Configuration

### Claude Code
To connect Claude Code to your Citric MCP server:

```sh
claude mcp add citric --transport http http://localhost:5173/api/mcp
```

Replace `localhost:5173` with your Citric deployment URL if running remotely.

## Available Tools

All tools accept optional `from` and `to` parameters for time range filtering (e.g. `now-7d`, `now-24h`). Defaults to the last 7 days.

### Overview

| Tool | Description |
|------|-------------|
| `get_dashboard_overview` | High-level CI/CD health: total runs, success rate, avg/p95 duration, repositories. Best starting point. |
| `get_top_failures` | Top 5 failing jobs and top 5 failing workflows. |

### Runs

| Tool | Description |
|------|-------------|
| `list_runs` | Search runs with filters (repo, branch, conclusion, workflow, runId). Paginated. |
| `get_run_details` | Full run hierarchy — metadata, jobs, and steps for a given traceId. |
| `get_step_logs` | Retrieve logs for a specific step (traceId + jobName + stepNumber). |

### Workflows

| Tool | Description |
|------|-------------|
| `get_workflow_details` | Comprehensive workflow analysis: stats with period comparison, trends, failing jobs, failure reasons, cost, and recent runs. |

### Tests

| Tool | Description |
|------|-------------|
| `get_test_results_summary` | Test health overview: total/pass/fail/skip counts, per-package breakdown, slowest tests. |
| `get_flaky_tests` | Flaky test identification with failure rates and flakiness scores. Filterable by repo/branch/search. |
| `get_test_history` | Execution history of a specific test with runner-level flakiness analysis. |

### Costs

| Tool | Description |
|------|-------------|
| `get_cost_analysis` | Cost breakdown: total, by OS, by runner tier, daily trend, per-repo, per-workflow. |
| `get_runner_utilization` | Runner performance metrics: jobs, avg duration, success rate per runner label. |

### Performance

| Tool | Description |
|------|-------------|
| `get_performance_trends` | Duration (avg/p50/p95), queue time, and success rate trends over time. |
| `get_failure_analysis` | Failure patterns with sample traceIds, failure trends, and failures by repo. |

## Example Usage

Once configured, you can ask your AI assistant questions like:

- "What's the current CI/CD health?"
- "Which workflows are failing the most?"
- "Show me the flaky tests in the last 24 hours"
- "Why did the latest run on main fail? Show me the logs"
- "How much are we spending on CI runners?"
- "Are there any performance regressions in our pipeline?"

The assistant will automatically call the appropriate Citric MCP tools to answer these questions.
